# Contributing to widget-service

Thanks for helping improve widget-service. This is the human contributor's front
door — it points you to the right doc rather than repeating it. (AI agents start
from [AGENTS.md](AGENTS.md) instead.)

## Getting started

1. Set up the dev environment and build from source — see [docs/CODING.md](docs/CODING.md#building-from-source).
2. Run the tests and learn the merge gates — see [docs/TESTING.md](docs/TESTING.md).
3. Branch, commit, and open a PR — see [docs/CHANGE-WORKFLOW.md](docs/CHANGE-WORKFLOW.md).

## Before you open a PR

- Lint and tests pass (commands live in [docs/CODING.md](docs/CODING.md) and [docs/TESTING.md](docs/TESTING.md)).
- Your change respects the architectural boundaries described in [docs/CODING.md](docs/CODING.md).

## Where things live

- Using the product (install, config, API): [README.md](README.md)
- Architecture and finding your way around: [docs/OVERVIEW.md](docs/OVERVIEW.md)
- Cutting a release: [docs/RELEASING.md](docs/RELEASING.md)

Open an issue before a large change so we can agree on the approach first.
